# Decisions and Open Questions

## Confirmed decisions

- The public refill calculator must be open without a password.
- Operational functions must be password protected.
- The customer cart must combine refills, Pantry Ready goods, glass, and other merchandise.
- Initial payments: cash, check, and Venmo.
- Common Good-provided refill containers will be glass only.
- Customers may bring suitable containers made from other materials, including plastic.
- Customer accounts are optional.
- Saved containers should retain tare weights and make repeat refills simpler.
- Pantry Ready products are a core sales format, not merely an emergency option.
- Pantry Ready inventory should also support vendor markets.
- Glass Collection is its own shopping step/category.
- Physical display concept: Pantry Ready above, refill at the main level, recommended glass below.
- The AI studio must use stored brand rules, approved labels, and product assets.
- Exact label text should be composited from approved artwork rather than regenerated by image AI.
- The app should be a PWA/mobile-friendly web app first.
- Sales tax should be configurable by location, with Alvo, Nebraska as the initial base location.
- Customer-facing prices should include sales tax rather than showing tax as a separate checkout line.
- The current app should use an 8% baked-in sales tax assumption for Alvo, Nebraska until exact rates are verified.
- Sales tax should apply to all current product categories because Common Good is not primarily selling food items.
- Supported refill weight units should include ounces and grams.
- Cash payments will not provide change.
- Each checkout should generate an easy-to-read payment code that links the cart to the payment envelope, check memo, or Venmo description.
- Every payment method should require a payment code for bookkeeping.
- The app does not need a payment processor while Common Good accepts only cash, check, and Venmo without automatic payment confirmation.
- Sales should be recorded in an internal ledger with original amount, rounded amount, entered/found amount, variance, notes, and status.
- Pantry Ready should be priced as its own format, with packaging, label, labor, waste buffer, and margin included in that format's COGS and retail price.
- Refill pricing should remain product-only, with purchased Common Good glass charged separately.
- AI Marketing Studio has been reset to a simple rebuild point based on the first iteration.
- For now, the visible Marketing Studio should keep only post focus, caption depth, buying lenses, one scene/goal text box, and a plain caption draft output.
- The first rebuilt image feature should generate a brand-matched image prompt that can be pasted into another AI platform. Actual in-app image creation can be added later after the prompt direction is working.
- Image prompts should be editable before image generation is requested so the owner can refine the scene, brand details, and exclusions first.
- The rebuilt Marketing Studio should move toward a one-step post kit: caption, one editable image prompt, and approved-label instructions generated together.
- Marketing output should elevate a level-1 idea into a level-10 concept by forming a point of view, visual metaphor, and stronger message instead of echoing the exact scene back to the owner.
- Image prompts should suggest the best specific image contents for the concept, including focal point, objects, placement, background, mood, and supporting details, instead of using the owner's rough goal as the final scene direction.
- Caption drafts and image prompts should display as review cards with pencil edit buttons, not always-open editing boxes.
- Approved post kits should be saved in a searchable prompt bank with filters and a used checkbox so ideas can be banked and returned to later.
- Prompt bank filters should include all-text search, caption-only keywords, used status, post focus, product/context, and buying lens.
- Each banked prompt kit should have compact upper-right actions to edit, copy, or delete the saved idea.
- Each banked prompt kit should include a second collapsed strategy drawer for CTA, ad spend notes, target demographics, sales psychology, and campaign details.
- When the product dropdown is absent, Marketing Studio should infer a mentioned product from the scene/goal text and make the caption and image prompt specific to that product.
- Marketing Studio should maintain product-specific social profiles for every product in the Products list, not only Room & Linen Spray, so buying lenses change copy and image direction across the full catalog.
- Product setup should eventually move to one owner intake screen with all relevant fields: product name, department/category, active/inactive status, materials, recipe, labor, waste, packaging, label assets, suggested price, actual price, refill format, Pantry Ready format, locations, inventory split, and social media aspects. New products should stay inactive until the owner marks them live.
- Product details should be entered once and auto-populate customer shopping screens, Product Notebook, COGS/pricing, inventory, location availability, and Marketing Studio.
- Add Product should save owner-created products locally, show inactive products only in the Workroom, and show active products on customer-facing refill, Pantry Ready, and request screens based on selected formats.
- Product Notebook tabs should be editable so owners can update status, formats, locations, recipes, COGS, prices, packaging, labels, and marketing notes as supply costs and availability change.
- COGS calculator should pull recipe lines into costable ingredient rows. Each row should capture recipe amount/unit, purchased container cost, purchased container size/unit, supplier, purchase link, and notes.
- COGS should also capture time to produce, labor rate/cost, packaging cost, label cost, optional packaging photo/link, waste buffer, and target margin.
- Formula-driven values, such as labor cost calculated from production time and hourly rate, should display as read-only calculated results rather than editable fields.
- Batch records should capture date, yield, bulk refill amount, Pantry Ready count, notes, and the calculated COGS basis used at the time.
- Refill pricing should be based on the bulk restock basis, while Pantry Ready pricing should add packaging, label, labor, and waste/margin assumptions.
- Pantry Ready products need size variants because one product may sell in multiple formats, such as 1 oz jar and 2 oz jar, with different retail prices and packaging costs.
- Image prompts should reference approved label/logo assets from the owner Workroom when available. AI should not invent new Common Good label text, logos, product names, poster text, or appliance text.
- Removed from the current rebuild: Content Bank, Brand Brain, product picker, platform, content type, hashtags, suggested audio, trend notes, account/location tags, ad angles, similarity warnings, and saved brief management.
- Cash and Venmo checkout totals should round down to the nearest dollar to avoid change and random Venmo amounts.
- Check checkout totals may remain exact unless later changed.
- Variance should be calculated from the rounded expected amount, not the original pre-rounding amount.
- Owner reporting should support Common Good overall data and location-specific splits.
- Owner reporting should support date-range filtering and filters such as location, product, product format, and payment method.
- Locations should be owner-managed. Adding a location should capture relevant settings such as products offered and sales tax rate.
- Saved containers can remain device-only for the current static app.
- Later customer accounts should be phone-number based rather than email based.
- Basic jars such as mason and Ball jars should be tracked as inventory counts.
- Unique glass pieces should have their own IDs.
- One-of-a-kind glass should be removed from inventory after purchase.
- If a unique Common Good jar is purchased, excluding basic mason/Ball-style jars, it should automatically be added to My Containers.

## Working assumptions

- Cash totals may be rounded down to the nearest dollar for the customer, but the business needs a pricing or margin buffer elsewhere to compensate.
- Payment codes should be short, memorable, and brand-appropriate, but not necessarily obviously biblical to customers.
- Venmo handle and QR code are not final yet; the app should ask the owner to enter verified business Venmo details before using Venmo checkout.
- Tare and filled weights may begin as manual entry while the scale workflow is still being decided.
- Sales Ledger discrepancies are recorded for balancing and review, not necessarily for customer follow-up or enforcement.
- Pantry Ready packaging and labor should not appear as a customer-facing surcharge; customers should see a single Pantry Ready price.
- Saved container screens should warn customers that clearing browser cookies/storage can erase saved jars and require re-taring.
- Glass inventory should eventually distinguish basic repeatable inventory from unique one-of-a-kind pieces.
- Jars should only be entered into sellable glass inventory if they have sealing lids. Airtight or gasketed requirements can be decided later.
- Discrepancy reports should help reveal whether variance balances out over time.
- Donations or intentional overpayments should be noteable and reportable, such as a donation to fill the jar shelf.

### Refill station
- Manual weight entry versus connected scale
- Whether pounds or fluid ounces are needed in addition to ounces and grams
- Whether liquid refills use weight, volume, or both
- Label printer model and label sizes
- Whether each Common Good glass piece uses QR, barcode, or manual search
- Mobile requests should primarily be Pantry Ready requests, not customer-container refills. This makes delivery easier because Common Good can keep 2-3 of each Pantry Ready item in a trunk box and fulfill without bringing the full refill setup.
- The customer-facing QR/magnet should simply open the Pantry Ready request page. That form should collect name, phone, Pantry Ready products, packaging/add-on notes, timing, and delivery or market pickup notes.
- Pantry Ready requests should show that the delivery fee is unknown until Common Good confirms scheduling.
- Pantry Ready requests should text both Alaina and Kaelea, and also land in the backend request log.
- Pantry Ready request forms should ask whether the customer wants text updates. The current static app records the opt-in and provides owner text links; a hosted version needs an SMS provider before sending automatic texts.
- If a customer has used Pantry Ready delivery from the same device before, their delivery/address notes should prefill so repeat requests are faster.
- Owner login password changed to `Refillery2026!`; the app should remember owner access on the same device for 90 days. A hosted version should replace this with real authentication.
- Backend Pantry Ready request log should include simple checkmarks for Scheduled and Delivered.
- Customer-container refill requests may be a future or exception workflow because they require bringing more inventory and reduce the chance for "since you are here" add-ons.
- Winter market workflow may rely on prepackaged mason jars / Pantry Ready stock first because customers may not know to bring jars or containers yet.
